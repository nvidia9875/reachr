# Reachr — 提出情報

> **あなたのデータに、誰が届くか。**
> 宣言（Terraform）と実際（GCP）のドリフトを可視化し、Gemini が塞ぐ。

## 概要（そのまま貼れる版）

Reachr は、GCP プロジェクトの中で **データストア（Cloud SQL・GCS など）に到達できる経路**を、ネットワークと IAM の両面からマッピングするツールです。

肝は **ドリフト検出**。Terraform（=あるべき姿）と、実際のデプロイ状態（Cloud Asset Inventory）を突き合わせ、**コードには無いのに本番でデータに届いてしまう経路**——障害対応でコンソールから開けたまま戻し忘れた DB、後付けで DB 権限を得たツール——を炙り出します。見つかった経路は **Gemini（Vertex AI）** が「なぜ危険か」を説明し、それを塞ぐ **Terraform パッチを生成**。PR では **CI が新しい到達経路を検出したらビルドを落とし**、経路をコメントします。

KDDI をはじめ、個人情報を溜め込む DB／バケットへの露出が実運用の中でじわじわ生まれる——その「気づけない露出」を、事故になる前に地図にします。

## 課題

インフラはリリース時はキレイでも、何年も運用するうちに現実がコードから乖離します。障害対応で `0.0.0.0/0` を開けたまま、分析ツールの SA に DB 権限が後付けされ、バケットが一時的に公開されたまま——こうした **コードの外で起きるドリフト**は、既存の IaC スキャンでは原理的に見えません。そして届く先が個人情報だと、そのまま漏洩事故になります。

## できること

1. **アタックサーフェス・マップ** — Internet → Edge → WAF → Compute → Data の到達グラフを可視化。危険な経路は赤く発光。
2. **ドリフト検出** — 宣言（Terraform）vs 実際（GCP）を diff し、本番だけに存在する到達経路を特定。
3. **Gemini による説明 & 修正** — 経路の危険性を自然言語で説明し、塞ぐ Terraform パッチを生成。自然言語クエリ（「インターネットから DB に届く経路ある？」）にも回答しハイライト。
4. **CI ゲート** — PR で新しい到達経路が生えたらチェックを落とし、経路を PR コメント＋ジョブサマリに出力（アタックパスの回帰テスト）。
5. **自律修復エージェント（`reachr agent`）** — ドリフトを検出 → Gemini が各経路を推論 → 塞ぐ Terraform パッチを自動生成し、PR 化まで用意する「検出 → 判断 → 修復」ループ。

## 3コンセプト対応（このハッカソンの評価軸）

| コンセプト | Reachr での実践 |
|---|---|
| **つくる** | Google Cloud の **Gemini（Vertex AI）** を中核に、到達経路を説明・修正する自律エージェントを実装 |
| **まわす** | **GitHub Actions**（`reachr ci`）で PR ごとにアタックパスを回帰テストし、継続的にセキュリティを維持 |
| **とどける** | **Cloud Run** へデプロイし、スケーラブルな本番環境で提供（`deploy.sh` 一発） |

## デモ動線（約2分）

1. マップを開く（ACTUAL）。DB が **EXPOSED（赤）** で、赤い経路が全チェーンを飛び越えてデータへ直達。
2. トグルを **DECLARED** へ。赤い経路が消え、DB が **PRIVATE（緑）** に。「コードはキレイ、でも現実は違う」を対比で見せる。
3. 明細をクリック → **Gemini が危険性を説明し、Terraform 修正パッチを生成**（コピー可）。
4. 「can anyone reach my DB from the internet?」と質問 → 該当経路がハイライト。
5. PR 上では **CI が FAIL** し、3 本の新規到達経路をコメント。

## 使用技術 → 開発要件マッピング

| 要件 | 充足 |
|---|---|
| **① GCP 実行プロダクト（必須）** | **Cloud Run** — Reachr の Web/API を Cloud Run で稼働（`Dockerfile` + `deploy.sh`） |
| **② GCP AI 技術（必須）** | **Vertex AI 経由の Gemini** — 経路の説明・Terraform 修正パッチ生成・自然言語クエリ |
| 対象データ源 | **Cloud Asset Inventory**（実際の状態）／`terraform show -json`（宣言） |
| 実装 | TypeScript（決定論グラフ）、依存ゼロの Web ビジュアライザ、GitHub Actions（`reachr ci`） |

> **設計方針**：グラフの真実（何が何に届くか）は **決定論コードで生成**し、LLM には判断させません。Gemini は「説明・修正・対話」だけを担当。ハルシネーションしないセキュリティ判定を担保しています。

## 差別化

同カテゴリの多くは「実行時のインシデントを AI エージェントが自動復旧する（DevOps/SRE）」系です。Reachr は軸が異なります：

- **予防（shift-left）× 設定ベース** — 事故る前に、設定から到達経路を塞ぐ。
- **データ到達に一点特化** — 「クラウンジュエルに誰が届くか」だけを深く。
- **決定論 × Gemini** — 判定は決定論、説明と修正は Gemini。
